OrthoMCL(v2.0.9) 使用介绍

发表于 2016-04-01   |   分类于 软件   |  
安装

软件下载后直接解压即可，截止至2016年3月最新的版本为v2.0.9。

环境配制
Linux

OrthoMCL只在Linux上测试过，且MCL程序只有Linux/Unix版本。

BLAST

NCBI BLAST支持tab分割的output格式。

Database

使用关系数据库虽然有点不方便，但加快运行速度，提高了程序的稳定性。数据库可以使用Oracle和MySql，本文使用MySql做为数据库测试。

Perl

标准perl程序和DBI库

orthoMCL软件安装服务器及路径：194@/data/biosoft/orthomcl-v2.0.9；
测试文件所在路径：/data/biosoft/orthomcl-v2.0.9/linsi/test/all路径下所有fasta格式文件；
配制文件路径：/data/biosoft/orthomcl-v2.0.9/orthomcl.config；
操作路径：/data/biosoft/orthomcl-v2.0.9/linsi/test/；
测试所需bin路径：/data/biosoft/orthomcl-v2.0.9/linsi/bin
注：步骤2-7写成的sh在194@/data/biosoft/orthomcl-v2.0.9/linsi/bin/sh_orthomcl_1，步骤11-18写成的sh在194@/data/biosoft/orthomcl-v2.0.9/linsi/bin/sh_orthomcl_2

注：测试之前需安装好数据库，MCL并配制好orthoMCL（如orthomcl.config）；
测试过程：

  1. pep源文件后缀改为fasta: perl /data/biosoft/orthomcl-v2.0.9/linsi/bin/batch_rename.pl /data/biosoft/orthomcl-v2.0.9/linsi/test/all
  2. 合并待测试fasta文件：cat all/*.fasta > all.fasta
    生成all.fasta文件
  3. pep文件转换为orthmcl所要求的文件：/data/biosoft/orthomcl-v2.0.9/bin/orthomclAdjustFasta hsa /data/biosoft/orthomcl-v2.0.9/linsi/test/all.fasta 1
    （has是自己命名的种名，1 表示将在你的每个蛋白名前加上种名，并且用|隔开）生成hsa.fasta文件
  4. 在test目录新建文件夹compliantFasta：mkdir compliantFasta
  5. 移动文件：mv hsa.fasta compliantFasta/
  6. 改写的蛋白文件进行过滤：/data/biosoft/orthomcl-v2.0.9/bin/orthomclFilterFasta /data/biosoft/orthomcl-v2.0.9/linsi/test/compliantFasta/ 10 20
    （10表示pep序列长度不低于10，20表示stop coden所占的百分比不高于20%，生成文件goodProteins.fasta和poorProteins.fasta）
  7. BLAST建蛋白库：formatdb -i goodProteins.fasta -p T
    生成文件goodProteins.fasta.phr, goodProteins.fasta.pin, goodProteins.fasta.psq
  1. 将前两步生成的文件拷贝到服务器70@/share/data/wanghy/linsi/project/2015/orthomcl_blastall/下面，在orthomcl_blastall下执行：rsync -avz wanghy@124.16.144.194:/data/biosoft/orthomcl-v2.0.9/linsi/test/goodProteins* .
  2. 为MCL提供相似矩阵：执行perl /share/data/wanghy/linsi/bin/orthomcl_blastall_bin/search_by_blast.pl –input /share/data/wanghy/linsi/project/2015/orthomcl_blastall/goodProteins.fasta –mode blastp –cpu 100 –database /share/data/wanghy/linsi/project/2015/orthomcl_blastall/goodProteins.fasta –outdir /share/data/wanghy/linsi/project/2015/orthomcl_blastall/abc/，在abc目录下生成goodProteins.fasta.blast
    备选方案（此法无需更换服务器，无需执行步骤7和8，但耗时长，不建议用此法做）：blastall -p blastp -i goodProteins.fasta -d goodProteins.fasta -m 8 F -b 1000 -v 1000 -a 2 -o all_VS_all.out.tab
    （生成文件all_VS_all.out.tab）
  3. 将上一步生成的文件goodProteins.fasta.blast转移到194@/data/biosoft/orthomcl-v2.0.9/linsi/test/，在194@test/执行rsync -avz wanghy@124.16.144.70:/share/data/wanghy/linsi/project/2015/orthomcl_blastall/abc/goodProteins.fasta.blast .
  1. 将goodProteins.fasta.blast改名，执行：mv goodProteins.fasta.blast all_VS_all.out.tab
  2. blast比对结果解析：/data/biosoft/orthomcl-v2.0.9/bin/orthomclBlastParser all_VS_all.out.tab /data/biosoft/orthomcl-v2.0.9/linsi/test/compliantFasta/ >> /data/biosoft/orthomcl-v2.0.9/linsi/test/similarSequences.txt
    （使用之前设置好的阈值等参数，设定在orthomcl.config文件；注：得删除中间结果文件，否则会报错；输出文件similarSequences.txt）
  3. 解析结果导入到数据库中：/data/biosoft/orthomcl-v2.0.9/bin/orthomclLoadBlast /data/biosoft/orthomcl-v2.0.9/orthomcl.config /data/biosoft/orthomcl-v2.0.9/linsi/test/similarSequences.txt
  4. 进行pairs的运算：/data/biosoft/orthomcl-v2.0.9/bin/orthomclPairs /data/biosoft/orthomcl-v2.0.9/orthomcl.config /data/biosoft/orthomcl-v2.0.9/linsi/test/orthomcl_pairs.log cleanup=no
  5. 处理数据库中pairs表：/data/biosoft/orthomcl-v2.0.9/bin/orthomclDumpPairsFiles /data/biosoft/orthomcl-v2.0.9/orthomcl.config
    （输出文件为mclInput）
  6. MCL，聚类：mcl /data/biosoft/orthomcl-v2.0.9/linsi/test/mclInput –abc -I 1.5 -o /data/biosoft/orthomcl-v2.0.9/linsi/test/mclOutput
    （输出文件为mclOutput）
  7. 结果转换成group.txt：/data/biosoft/orthomcl-v2.0.9/bin/orthomclMclToGroups my_prefix 1 < mclOutput > groups.txt
    （myprefix 指定在groups.txt中每个家族的前缀，如：GF 则在groups.txt中，每个家族以GF_开始，1表示家族从1开始编码；生成的groups.txt可将相似基因聚类）
  8. 获得矩阵：perl /data/biosoft/orthomcl-v2.0.9/linsi/bin/ortholog_matrix.pl groups.txt all/strain_name matrix_out
流程代码（perl）
use DBI;
die "usage: $0 <rawdata_path> <output_path>\n" if (@ARGV !=2);
my $db_name="orthomcl";        #数据库名字
my $db_host="localhost";    #本地操作直接localhost
my $db_port="3306";            #mysql端口(默认是3307)
my $username="root";        #mysql用户名
my $password="password";    #mysql密码
my $dsn="dbi:mysql:database=${db_name};hostname=${db_host};port=${db_port}";
my $drh=DBI->install_driver("mysql");
my $bin_path="/data/biosoft/orthomcl-v2.0.9/bin";
my $orthomcl_config="/data/biosfot/orthomcl-v2.0.9/orthomcl.config";
#删除旧的数据库（残留的同名数据库可能会有影响）
if ($rc=$drh->func("dropdb",$db_name,$db_host,$username,$password,"admin")) {
    print "drop database `",$db_name,"` successfully!\n";
}
#新建数据库
$rc=$drh->func("createdb",$db_name,$db_host,$username,$password,"admin")or die "failed to create database",$db_name,"!\n";
print "create database `orthomcl` successfully!\n";
#使用config文件对mysql进行配置，即对config文件里指定orthomcl数据库建立一些表
system "$bin_path/orthomclInstallSchema $orthomcl_config $ARGV[1]/install_schema.log";
open SH, ">$ARGV[1]/run.sh";
#将pep格式文件合并成all.fasta
print SH "cat $ARGV[0]/* > $ARGV[1]/all.fasta\n";
#all.fasta转换成orthomcl需要的格式，输出文件为hsa.fasta
print SH "$bin_path/orthomclAdjustFasta hsa $ARGV[1]/all.fasta 1\n\n";
#新建文件夹compliantFasta，并转移文件hsa.fasta至此文件夹
print SH "mkdir $ARGV[1]/compliantFasta\n";
print SH "mv $ARGV[1]/hsa.fasta $ARGV[1]/compliantFasta/\n";
#过滤蛋白文件，pep序列长度不低于10, stop coden所占的百分比不高于20%
print SH "$bin_path/orthomclFilterFasta $ARGV[1]/compliantFasta/ 10 20\n";
#建库并blast为mcl提供相似矩阵
print SH "formatdb -i $ARGV[1]/goodProteins.fasta -p T\n";
print SH "blastall -p blastp -i $ARGV[1]/goodProteins.fasta -d $ARGV[1]/goodProteins.fasta -m 8  F -b 1000 -v 1000 -a 2 -o all_VS_all.out.tab";
#将blast结果进行解析
print SH "$bin_path/orthomclBlastParser all_VS_all.out.tab  $ARGV[0]/compliantFasta/ >> $ARGV[0]/similarSequences.txt\n";
#将解析好的结果导入到mysql中
print SH "$bin_path/orthomclLoadBlast $orthomcl_config $ARGV[0]/similarSequences.txt\n";
#对数据库中SimilarSequences表进行pairs运算，并在数据库中产生三个表
print SH "$bin_path/orthomclPairs $orthomcl_config $ARGV[0]/orthomcl_pairs.log cleanup=no\n";
#处理数据库中pairs表，生成mclInput文件和一个文件夹pairs
print SH "$bin_path/orthomclDumpPairsFiles $orthomcl_config\n";
#对上一步结果进行mcl操作，开始聚类
print SH "mcl $ARGV[0]/mclInput --abc -I 1.5 -o $ARGV[0]/mclOutput\n";
#将上步操作结果转换成groups.txt
print SH "$bin_path/orthomclMclToGroups my_prefix 1 < mclOutput > groups.txt\n";
close SH;
system "sh $ARGV[1]/run.sh";
orthomcl.config配制文件
# this config assumes a mysql database named 'orthomcl'.  adjust according
# to your situation.
dbVendor=mysql
dbConnectString=dbi:mysql:orthomcl:localhost:3306
dbLogin=root
dbPassword=biosoft
similarSequencesTable=SimilarSequences
orthologTable=Ortholog
inParalogTable=InParalog
coOrthologTable=CoOrtholog
interTaxonMatchView=InterTaxonMatch
percentMatchCutoff=50
evalueExponentCutoff=-5
oracleIndexTblSpc=NONE
